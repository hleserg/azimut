## 1. Архитектура: источник правды

**`workspace.dsl`** в корне репо — единственный источник правды для статичных C4-диаграмм
(Context / Container / Component). Фиксируется ADR 0034.

- Просмотр локально: `docker compose --profile diagrams up -d structurizr` (остановить: `docker compose --profile diagrams down`)
- Открыть `http://localhost:8080`
- Перед изменением `workspace.dsl` — прочитать `_source/specs/c4/` (C4-спека).
- Mermaid `sequenceDiagram` — только для §6 Runtime View (он лучше читается в git diff).
  Статичные C4 в Mermaid **не создавать** (нарушает ADR 0034).

## 2. ADR — архитектурные решения

**Где живут:** `docs/architecture/adr/<подпапка>/NNNN-kebab-title.md`

**5 подпапок и их темы:**

| Подпапка | Тема | Диапазон |
|---|---|---|
| `anti-hallucinations/` | Поведенческий контракт: Р1–Р7, П1–П3 | 0001–0010 |
| `foundation/` | Фундамент: форк, имя, стек, клиент, модель | 0011–0023 |
| `code-processing/` | Обработка кода 1С: чанкинг, резолв, роутинг | 0024–0027 |
| `tooling/` | Инструментарий и процесс | 0034 |
| `open/` | Открытые/proposed: Sentry×AGPL, мульти-аренда | 0028–0033 |

**Нумерация:** сквозная `0001+` по всем подпапкам. Тема определяется подпапкой, не номером.
**Шаблон:** `docs/architecture/adr/template.md` (не ADR, в нумерацию не входит).
**Текущий max:** 0034. Следующий — 0035.

**Команда создания нового ADR:**

```bash
# Linux / macOS / WSL
./scripts/new-adr.sh <подпапка> <kebab-title>
# Пример:
./scripts/new-adr.sh anti-hallucinations new-detection-rule

# Windows PowerShell
.\scripts\new-adr.ps1 <подпапка> <kebab-title>
```

Скрипт: читает все `adr/*/NNNN-*.md`, берёт max номер, инкрементирует,
копирует `template.md`, подставляет дату и название.

## 4. Индекс ADR (09-architectural-decisions.md)

Файл автогенерируется скриптом — **не редактировать вручную** таблицу между маркерами:

```bash
./scripts/update-adr-index.sh        # Linux / WSL
.\scripts\update-adr-index.ps1      # Windows
```

Скрипт парсит фронтматтеры всех `adr/*/NNNN-*.md`, обновляет таблицу
между маркерами `<!-- ADR-INDEX:START -->` и `<!-- ADR-INDEX:END -->`.

## 6. Definition of Done (DoD)

Задача **не закрыта**, если не выполнено:

1. **`workspace.dsl` обновлён** — если изменена архитектура (новый контейнер, компонент,
   связь). Рендер проверяется через Structurizr (`structurizr/structurizr`, см. §1).
2. **ADR создан или обновлён** — любое архитектурное решение фиксируется в MADR-формате.
   Команда: `./scripts/new-adr.sh <подпапка> <kebab-title>`.
3. **Глава 13 обновлена** — если добавлен новый инфраструктурный элемент.
4. **Индекс пересобран** — `./scripts/update-adr-index.sh` запущен после создания ADR.

## 7. Правило источников (ADR 0023)

При написании любого утверждения в документации:

- `✅ проверено: <файл/url>` — если утверждение подтверждено источником
- `⚠️ предположение` — если источника нет, но утверждение правдоподобно

Агент **не должен** писать непроверенное утверждение без маркера `⚠️ предположение`.

## 10. Статус-протокол агентов

- **Взял задачу** → переведи Linear-issue в `In Progress`.
- **PR готов** → переведи в `In Review`. `Done` ставит **только Сергей**.
- При делегировании sub-агентам: статус меняется **в момент делегирования / возврата**,
  не батчем в конце.
