## Типы файлов и читаемость
### Читаются напрямую
- `.ini` / `.def` / `.stp` — текстовые конфиги, кодировка Windows-1251
### Не читаются без парсера (проприетарный формат)
**`Data.cab`** (магия `ISCF`) — основной архив с контентом. Внутри:
- HTML-файлы с документацией (`SRC/INDEX.HTM`, `*.HTM`)
- Изображения PNG и JPEG (их сигнатуры видны прямо в байтах файла)
- Сжатие: предположительно LZX или deflate, стандартный zlib не подошёл
- Стандартные инструменты (`7z`, `cabextract`) формат **не поддерживают** — это не Microsoft MSCF CAB
**`Data.dir`** (магия `CFHD`) — индекс/оглавление к кабинету. Через `strings` виден список путей: `SRC/_IMG/*.PNG`, `SRC/*.HTM` и т.д.
**`.dat`**** / ****`.dbs`**** / ****`.dbv`** — проприетарные B-tree таблицы ИТС (оглавление, атрибуты, классы). Бинарные, нечитаемые.
**`.lex`**** / ****`.voc`** — морфологические словари. Бинарные.
## Вывод: конвертировать один раз
### Почему не работать с ISCF напрямую
- Нет готовых библиотек — парсить проприетарный бинарник при каждом RAG-запросе хрупко и медленно
- Формат не документирован, может отличаться между версиями ИТС
- Никакой LangChain / LlamaIndex loader под него не написан
### Рекомендуемый пайплайн
```javascript
Data.cab + Data.dir
        ↓  (один раз, Python-скрипт ~300 строк)
HTML + images (статические файлы, структура SRC/ сохранена)
        ↓  (при индексации)
Markdown-чанки с inline ![]() для картинок
        ↓
Embeddings → Vector store
```
**Целевой формат после конвертации: HTML с локальными путями к картинкам.**
Картинки остаются ровно на своих местах через `<img src="SRC/_IMG/...">` — ничего не теряется.
### Что нужно написать
Python-парсер ISCF:
1. Читает `Data.dir` (CFHD-формат) — получает индекс файлов с именами и офсетами в кабинете
2. Читает `Data.cab` (ISCF-формат) — распаковывает блоки по офсетам
3. Сохраняет HTML + изображения в файловую систему с оригинальной структурой `SRC/`
После этого стандартный HTML-парсер (BeautifulSoup) режет на чанки по заголовкам, картинки идут inline.
### Оценка объёма
- Исходник: \~300 МБ (сжато)
- После извлечения: ориентировочно \~150–200 МБ HTML + картинки
- Разделов: 15 (ElReps, Metod8Dev, V8DevGloss и др.) — каждый конвертируется независимо
