## Зафиксированные решения

* **Язык:** Python ≥3.11, uv. **MCP:** FastMCP. **Разговорная модель:** облачный Claude (+нативное зрение для скринов — мультимодальные эмбеддинги для этого НЕ нужны).
* **Эмбеддинги (основное):** BGE-M3 локально через **родной SDK FlagEmbedding** — dense+sparse+ColBERT одним вызовом, гибридный скоринг из коробки. OpenAI — опц. фолбэк.
* **Реранкинг:** **Cohere Rerank v4** через API (грант $1000; \~$2/1000 поисков). BGE-reranker-v2-m3 — локальный фолбэк без ключа.
* **Картинки:** OCR+текст основным; **Cohere Embed 4** (мультимодал) — кандидат на обкатку (задача 12).
* **LLM-судья:** Claude (вне retrieval-конвейера). Cohere Command — опц. второй голос.
* **Векторное хранилище: Qdrant (embedded режим)** — основной выбор. Обоснование: нативно хранит dense + sparse в одной коллекции (ровно выход BGE-M3); поддерживает несколько именованных векторов на точку (наши «несколько представлений на чанк» для shadow-эксперимента, задача 11); запускается в процессе Python БЕЗ отдельного сервера (критично для нетехнического пользователя — маме не надо ставить вторую сущность). Пересмотр только если упрёмся на реальном объёме данных.

## Почему не Milvus / pgvector

* **Milvus Lite** — нативный гибрид bge-m3 есть, но Lite слабее по фичам и капризнее на Windows (а у пользователей Windows). Избыточен для однопользовательского локального сценария.
* **pgvector** — требует поднять и держать PostgreSQL = лишняя сущность для нетехнического пользователя; гибрид руками сложнее.
