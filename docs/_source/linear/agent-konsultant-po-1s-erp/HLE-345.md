## Что проверить

### MCP-сервер подключён к Claude Desktop

- [ ] Добавь сервер в `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "1c-consultant": {
      "command": "uv",
      "args": ["run", "python", "-m", "src.server"],
      "cwd": "/путь/к/1c-consultant"
    }
  }
}
```

- [ ] Перезапусти Claude Desktop
- [ ] В интерфейсе Claude появился значок инструментов (молоток) — нажми, убедись что инструменты 1С-консультанта видны

### Онбординг (задача 10)

- [ ] Открой новый чат с Claude — агент должен сам начать онбординг (спросить путь к конфигурации)
- [ ] Пройди онбординг: укажи пути к данным
- [ ] Агент предложил добавить личные документы — откажись (проверь что не настаивает)
- [ ] Агент задал вопрос про отчёты об ошибках — ответь как хочешь
- [ ] Конфиг сохранился: `cat data/config.json` (или аналог) — пути прописаны

### Индексация через CLI (задача 8)

- [ ] Запусти полную индексацию: `uv run python -m src.cli index --all`
- [ ] Прогресс отображается (не висит молча)
- [ ] После завершения агент в Claude об этом знает — спроси его «что проиндексировано?»

### Базовые инструменты работают

- [ ] Спроси Claude: «какой у меня статус индексации?» — должен вызвать `get_index_status` и ответить
- [ ] Спроси Claude: «найди что-нибудь про склады в 1С» — должен вызвать поисковый инструмент и показать результат

## Если что-то не работает

* **Инструменты не появились в Claude Desktop** → проверь путь в `claude_desktop_config.json`, перезапусти Claude полностью (не просто новый чат)
* **Агент не начинает онбординг** → возможно системный промпт не подключён или инструменты не отдают нужные описания — смотри логи сервера
* **CLI зависает на индексации** → Ctrl+C, смотри что последнее вывелось, ищи в Sentry трейс с ошибкой
* `get_index_status` **возвращает пустоту** → возможно сервер и CLI смотрят на разные пути к данным — проверь конфиг
