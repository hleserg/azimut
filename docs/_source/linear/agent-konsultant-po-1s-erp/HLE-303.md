Менеджер загрузки/выгрузки локальной модели BGE-M3 (через FlagEmbedding). Цель: модель в RAM только когда активна — не висит зря, но и не лагает на каждый запрос.

## Жизненный цикл

* **Ленивая загрузка + синглтон.** Модель НЕ грузится при старте сервера. Поднимается при первом обращении, живёт единственным экземпляром.
* **Прогрев на healthcheck.** `healthcheck` (задача 10) запускает загрузку в ФОНОВОМ потоке и СРАЗУ возвращает ответ (не блокирует). Пока агент здоровается — модель греется.
* **Выгрузка по таймауту простоя.** Не обращались N минут → выгрузить из RAM. Следующий healthcheck снова прогреет.
* **Режим индексации — исключение.** Во время массовой индексации модель резидентна весь прогон, таймаут не применяется.

## Конфиг

* `model.idle_timeout_minutes` (0 = держать всегда), `model.warm_on_healthcheck`, `model.always_resident`.

## Definition of done

* Холодный старт сервера не грузит модель (проверить по RAM).
* `healthcheck` инициирует фоновый прогрев, не блокируется на загрузку.
* После таймаута простоя RAM освобождается; следующий запрос/healthcheck перезагружает.
* Индексация удерживает модель весь прогон независимо от таймаута.
* Параметры из конфига.

## Примечание

Чем больше на Cohere (rerank/мультимодал — API), тем меньше резидентной RAM. Lifecycle касается только локального BGE.
