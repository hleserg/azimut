Периодическое (раз в месяц) обновление документации 1С: ненавязчиво предложить, провести по процедуре скачивания/разворачивания новой доки, переиндексировать. Использует механизм живых процедур (задача 17).

## Поведение (строго по требованиям)

* **Ненавязчивое предложение.** Агент сам не лезет с обновлением, а мягко упоминает при удобном случае: «вышли новые релизы 1С, заодно могли появиться новые материалы — обновим доки?». Частота — примерно раз в месяц, не чаще.
* **Процедура обновления — живая** (задача 17): `procedures/update_docs.md`. На старте — лучшее предположение (возможно, офлайн-ИС ИТС обновляется кнопкой в своём UI; возможно, надо перекачивать — НЕИЗВЕСТНО, уточнить первым реальным прогоном и дать агенту дописать процедуру + прислать админу).
* **Согласование переиндексации ЗАРАНЕЕ.** Перед запуском индексации агент предупреждает: база на время индексации будет неполной. Спрашивает, не нужно ли пользователю сейчас что-то узнать ДО того, как запускать (чтобы не остаться без данных в нужный момент).
* **Напоминания во время индексации.** Пока идёт переиндексация — при каждом обращении агент напоминает: часть данных временно не видна, пока не закончится (через get_progress / healthcheck, задача 10). По завершении — сообщить, что снова доступно всё.

## Definition of done

* Механизм ненавязчивого ежемесячного предложения работает (не чаще раза в месяц, не назойливо).
* Обновление идёт по живой процедуре; при расхождении с реальностью — чинится и шлётся админу (задача 17).
* Переиндексация согласуется с пользователем до запуска.
* Во время переиндексации агент напоминает о неполноте данных; по завершении сообщает о готовности.
