Кеш поисковых запросов двух уровней для снижения стоимости и латентности при повторяющихся или близких вопросах.

## Уровень 1: Exact-match cache (TTL-кеш)

Простой словарь `query_text → результат поиска` с TTL. Применяется для системных вызовов (healthcheck, get_progress) и идентичных запросов.

```python
from functools import lru_cache
import time

class ExactCache:
    def __init__(self, ttl_seconds: int = 3600):
        self._store: dict[str, tuple[Any, float]] = {}
        self.ttl = ttl_seconds
  
    def get(self, key: str) -> Any | None:
        if key in self._store:
            value, ts = self._store[key]
            if time.time() - ts < self.ttl:
                return value
            del self._store[key]
        return None
```

## Уровень 2: Semantic cache (по эмбеддингу запроса)

Кешируем результат поиска, если новый запрос семантически близок к уже отвеченному. Используем те же BGE-M3 эмбеддинги что уже есть в системе — дополнительных зависимостей нет.

```python
class SemanticCache:
    def __init__(self, threshold: float = 0.95, max_entries: int = 500, ttl: int = 3600):
        self.threshold = threshold  # из конфига
        self.max_entries = max_entries
        self.ttl = ttl
        self._entries: list[CacheEntry] = []  # (embedding, result, timestamp)
  
    def lookup(self, query_embedding: np.ndarray) -> Any | None:
        now = time.time()
        for entry in self._entries:
            if now - entry.ts > self.ttl:
                continue
            sim = cosine_similarity(query_embedding, entry.embedding)
            if sim >= self.threshold:
                return entry.result
        return None
```

При `max_entries` — вытесняем LRU.

## Что НЕ кешируем

* Первый вопрос в новой сессии (флаг reindex_needed мог измениться)
* Запросы о статусе индексации (всегда свежие данные)
* При включённом shadow-режиме (задача 11) — не кешируем, нужны реальные прогоны

## Телеметрия

Для каждого запроса логировать в Sentry:

* `cache_hit: true/false`
* `cache_type: "exact" | "semantic" | null`
* `cache_similarity: float` (для semantic)

Cache hit rate — важная метрика эффективности (задача 11).
