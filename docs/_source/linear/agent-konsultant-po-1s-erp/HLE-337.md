## Поддерживаемые форматы

Реализовать парсинг следующих форматов — все они должны читаться **надёжно и полностью**:

| Формат | Библиотека | Примечание |
| -- | -- | -- |
| `.txt`, `.md` | stdlib | Прямое чтение |
| `.pdf` (текстовый) | `pdfplumber` + `pymupdf` | Основной путь |
| `.pdf` (скан/картинка) | `pytesseract` / `easyocr` | OCR-фолбэк |
| `.docx` | `python-docx` | Включая картинки внутри — OCR |
| `.doc`, `.odt` | LibreOffice headless | Конвертация в docx → затем python-docx |
| `.pptx` | `python-pptx` | Текст слайдов + заметки |
| `.csv` | `pandas` | Содержимое как текст |
| `.xml` | `lxml` | С пометкой в метаданных: «структурированные данные» |
| `.json` | stdlib | Текстовая выдача ключей и значений |
| `.html`, `.htm` | `beautifulsoup4` | Извлечение текста без тегов |
| `.eml` | stdlib `email` | Тема + тело письма |

### LibreOffice headless — стратегия

Для `.doc` и `.odt` использовать LibreOffice в headless-режиме для конвертации в `.docx` во временный файл, затем читать через `python-docx`. Не парсить через LibreOffice API напрямую — так чище и предсказуемее.

LibreOffice — обязательная зависимость при наличии `.doc`/`.odt` файлов. Если LibreOffice не установлен — сообщить пользователю при сканировании таких файлов, что нужна установка.

### Форматы с advisory-статусом (не парсим, сообщаем)

`.xlsx`**,** `.xls` — при обнаружении агент сообщает:

> «Нашёл файл `[имя_файла].xlsx`. Если это описание процесса или регламент — сохрани его как `.docx`, я смогу его прочитать. Если это расчётная таблица — я консультант по 1С, а не по числам в Excel, так что она мне в любом случае не поможет 😄»

Файл записывается в реестр (задача 1a) со статусом `advisory`. Не ошибка, не успех — пользователю сказали что делать.

### Неподдерживаемые форматы

Всё что не входит в список выше — записывается в реестр (задача 1a) со статусом `unsupported`. При следующем запросе агент сообщает пользователю:

> «Файл `[имя]` не удалось прочитать — формат `[расширение]` не поддерживается. Поддерживаемые форматы: txt, md, pdf, docx, doc, odt, pptx, csv, xml, json, html, eml.»

## Рекурсивный проход

* Использовать `pathlib.Path.rglob('*')` для обхода папки любой глубины вложенности
* Фильтровать по расширению (регистронезависимо)
* Скрытые папки (начинаются с `.`) — пропускать

## Картинки внутри документов

* Для `.docx` с вставленными изображениями: вырезать изображения, прогнать через OCR, добавить текст к содержимому документа
* Для `.pdf` со смешанным содержимым (текст + картинки): сначала попытка `pdfplumber`, если страница возвращает мало текста — OCR этой страницы
* OCR-движок: `pytesseract` (первичный) + `easyocr` (фолбэк и для кириллицы)

## Healthcheck папки (интеграция с задачей 1a)

При каждом healthcheck:

* Рекурсивно сканировать папку с личными доками
* Сравнивать текущий состав файлов и их хеши (sha256) с реестром (задача 1a)
* Новые файлы → добавить в реестр как `pending`
* Изменённые файлы (хеш не совпадает) → пометить как `pending` (требуют переиндексации)
* Удалённые файлы → пометить как `deleted` в реестре, удалить чанки из Qdrant
* Если есть `pending` → выставить флаг `user_docs_reindex_needed: true` в стейте

Агент при следующем запросе пользователя, если флаг поднят, ненавязчиво сообщает: «Кстати, я заметил изменения в твоей папке с документами. Запустить переиндексацию?»

## Управление личными доками через агента

Пользователь может в любой момент:

* Указать/сменить путь к папке: «у меня теперь личные доки на `C:\sometrash\`» → агент вызывает `update_settings('user_docs_path', ...)`, предлагает сразу запустить индексацию
* Отключить: «не хочу давать доступ к своим документам» → `user_docs_enabled: false`, механизмы папки не запускаются
* Включить снова: «добавь мои документы» → агент спрашивает путь, включает

### Если пользователь не указал папку при онбординге

Флаг `user_docs_enabled: false` — всё молчит. Не напоминать и не предлагать самостоятельно. Только если пользователь сам попросит.

## Предупреждение о коммерческой тайне (интеграция с задачей 10)

При первой активации личных доков, если используется облачная модель (Claude, ChatGPT и т.п.):

> «Прежде чем начать — имей в виду: фрагменты твоих документов будут отправляться в облако для поиска ответов. Если в них есть коммерческая тайна или конфиденциальная информация — лучше сначала уточни у своего руководства, можно ли это делать. Готов продолжить?»

Флаг `privacy_warning_acknowledged: true` с датой — показываем один раз. При локальной модели — не показываем вообще.

## Инкрементальная индексация

* При переиндексации: файл не изменился (хеш совпадает) И последняя индексация успешна → пропустить
* Файл не изменился, но статус `error` → **переиндексировать** (прошлый раз сломался)
* Файл новый или изменился → индексировать
* Качество приоритетнее скорости: лучше лишний раз переиндексировать, чем пропустить изменение
