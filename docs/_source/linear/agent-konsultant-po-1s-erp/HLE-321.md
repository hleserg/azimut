# 0h: Изучить agent-playbook и перенести полезное в правила проекта

## Контекст

Репо [`hleserg/agent-playbook`](<https://github.com/hleserg/agent-playbook>) — коллекция находок и паттернов, которые агенты Atman накопили в процессе работы над разными проектами. Склонировано локально.

Задача: агент изучает **весь** этот проект и **весь** текущий проект `1c-consultant` (задачи, ADR, AGENTS.md, структуру, стек), затем вытаскивает из playbook всё, что реально применимо здесь — и устанавливает это в репо проекта в виде правил или рекомендаций.

## Что делать

1. **Изучить** `agent-playbook` **целиком** — все файлы, разделы, паттерны, находки.
2. **Изучить проект** `1c-consultant` **целиком** — задачи в Linear (все <issue id="0c3fafe6-14eb-4ee5-a258-da0c1f90c4c1">HLE-292</issue>..320), AGENTS.md, структуру репо, стек (Python/uv/FastMCP/Qdrant/BGE-M3/Cohere), поведенческий контракт агента, ADR.
3. **Отобрать применимое** — критерии отбора:
   * Паттерн решает реальную проблему, которая есть или вероятна в этом проекте
   * Не противоречит уже принятым решениям (ADR, AGENTS.md)
   * Конкретен и actionable, а не абстрактный совет
   * Не дублирует то, что уже есть в AGENTS.md или задачах
4. **Установить в репо** — в зависимости от типа находки:
   * Правила для агентов → добавить секцию или пункты в `AGENTS.md`
   * Рекомендации по стеку/архитектуре → добавить в `docs/` (новый файл или в существующий)
   * Паттерны тестирования/CI → в соответствующие конфиги или `docs/`
   * Если не ясно куда — в `docs/agent-notes.md` с объяснением
5. **Отчитаться в комментарии к этой задаче** (см. DoD)

## Definition of Done

- [ ] `agent-playbook` изучен полностью
- [ ] `1c-consultant` (репо + все Linear-задачи) изучен полностью
- [ ] Каждая перенесённая находка установлена в репо в подходящее место
- [ ] PR создан по стандартному git-workflow (ветка → draft → снять draft → зелёный CI → ревью → мердж)
- [ ] Комментарий к этой задаче содержит отчёт: для каждой взятой находки — **что взял**, **откуда** (файл/раздел в playbook), **почему применимо именно здесь**, **куда положил** в репо
- [ ] Для каждой находки из playbook, которую НЕ взял — краткое объяснение почему (не применимо, уже есть, противоречит ADR и т.п.) — одной строкой достаточно

## Примечания

* Качество отбора важнее количества. Лучше 3 действительно полезные штуки, чем 15 для галочки.
* Не переписывать AGENTS.md с нуля — только добавлять/уточнять.
* Если в playbook есть что-то крутое, но непонятно куда применить — упомянуть в комментарии отдельно, Сергей решит.
* Путь до склонированного `agent-playbook` уточнить у Сергея перед началом (или найти самостоятельно через стандартные пути).
