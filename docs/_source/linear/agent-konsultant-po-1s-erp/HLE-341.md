## Как проверить Sentry

```python
# Запусти локально после того как DSN прописан в .env
import sentry_sdk
from dotenv import load_dotenv
import os

load_dotenv()
sentry_sdk.init(dsn=os.environ["SENTRY_DSN"])
raise RuntimeError("КТ1: тестовое событие Sentry")
```

Должно появиться в Sentry через \~10 секунд. Если нет — смотри вкладку `Issues` → фильтр `is:unresolved`, и проверь что DSN скопирован без пробелов.

## Если что-то не работает

* **CI не зелёный** → смотри вкладку Actions в GitHub, читай лог упавшего шага
* **Devin не реагирует** → проверь что он добавлен в CODEOWNERS или как reviewer в настройках репо
* **Sentry не получает событие** → убедись что DSN в Secrets совпадает с тем что в Sentry → Project Settings → Client Keys
* **Branch protection не работает** → Settings → Branches → Branch protection rules → убедись что required checks прописаны
