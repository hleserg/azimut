CLI и оркестрация пайплайна. Сценарий: «склонировал → указал пути → индексация → подключил MCP → консультируешься». **Индексация обязана отдавать прогресс** — на нём строятся `get_progress` и опрос агентом.

## Что сделать

* CLI: `erp-agent index` (полная), `index --only <source>`, `reindex` (инкрементально), `serve`, `status`.
* Оркестратор: по всем конфигурациям и докам из конфига → парсеры (3, 4) → общий индекс.
* **Прогресс с состоянием:** процент + оценка оставшегося времени + текущий этап в разделяемое состояние (читает `get_progress`, задача 10). Оценка — по числу файлов/чанков и скорости первых N.
* **Автозапуск при save_config:** обновились пути → индексация в фоновом потоке, не блокируя MCP-ответы.
* Прогресс-бар/логи в CLI.
* Понятные ошибки: нет пути, пусто, битый XML — человекочитаемо.

## Definition of done

* Чистая машина: клон → install → пути → индексация → `serve` по README.
* `get_progress` во время индексации отдаёт корректные процент + оценку времени.
* `save_config` запускает индексацию в фоне, сервер отвечает на другие инструменты.
* `status` показывает состояние индекса.
* Инкрементальная переиндексация быстрее полной на неизменных данных.
