## Сценарии для проверки

Задай минимум 10 реальных вопросов из своей рабочей практики. Для каждого отмечай:

### Блок 1: Поиск по документации

- [ ] Вопрос на тему из ИТС (например: «как настроить ордерную схему склада?») — агент отвечает из источников, не выдумывает
- [ ] Вопрос на тему которой нет в доках — агент честно говорит «не нашёл», не галлюцинирует
- [ ] Вопрос про типовой объект конфигурации (реквизит, справочник) — агент находит в XML-выгрузке

### Блок 2: Поведение «спец, а не колл-центр»

- [ ] Задай расплывчатый вопрос («что-то с заказами не работает») — агент уточняет, а не выдаёт простыню наугад
- [ ] Задай вопрос не по теме 1С (например: «напиши мне письмо директору») — агент вежливо отказывает и объясняет зону своей ответственности
- [ ] Задай вопрос по которому нужно несколько источников — агент собирает из нескольких, а не останавливается на первом

### Блок 3: Личные документы (если добавил)

- [ ] Вопрос по теме регламента из личных доков — агент находит и цитирует
- [ ] Вопрос смешанный (ИТС + личный регламент) — агент комбинирует источники

### Блок 4: Технические проверки

- [ ] Агент не отвечает из памяти без поиска — в чате видно вызов инструмента перед ответом
- [ ] Ответ содержит ссылку на источник («согласно разделу X документации...»)
- [ ] Healthcheck во время сессии: спроси «всё ли в порядке с индексом?» — агент проверяет и отвечает

## Критерии провала (нужно исправлять до закрытия)

* Агент отвечает уверенно на вопрос которого нет в доках — **галлюцинация, критично**
* Агент игнорирует инструменты и отвечает из памяти — **нарушение поведенческого контракта**
* Агент не может найти ничего по базовым темам 1С ERP — **проблема с индексацией или поиском**
* Ответы занимают > 30 секунд — **проблема с производительностью**
