Перед тем как чанк уходит в Qdrant, он проходит через PII-фильтр. Цель — не допустить попадания персональных и чувствительных данных в облачную модель при поиске. Исходный файл на диске **не трогается** — маскируем только то что уходит в индекс и потом в контекст Claude.

## Что маскируем — российский корпоративный контекст

| ОГРН | 1027700132195 | 13 цифр |

### NER-распознаватель (контекстные сущности, spaCy)

Модель: `ru_core_news_sm` (\~45MB, CPU, F1≈0.95 на русском)

Детектирует:

* `PER` — ФИО (Иванов Иван Иванович, Петрова А.С.)
* `ORG` — названия организаций в контексте (ООО «Ромашка», ИП Сидоров)
* `LOC` — адреса и локации (ул. Ленина, д.5, кв.12)

### Что НЕ маскируем

* Названия типовых объектов 1С (Справочник.Контрагенты — это не PII, это метаданные)
* Названия полей и реквизитов конфигурации
* Примеры из официальной документации

## Стек

```
presidio-analyzer        # orchestration, regex + NER
presidio-anonymizer      # маскировка найденного
spacy + ru_core_news_sm  # NER для русского языка, CPU
```

Presidio поддерживает кастомные распознаватели через `PatternRecognizer` — туда идут все российские regex-паттерны.

## Поведение при нахождении PII

1. Маскируем в тексте чанка: `Иванов Иван Иванович` → `[PER]`, `+7 916 123-45-67` → `[PHONE_RU]`
2. Записываем в реестр (задача 1a): `pii_detected: true`, `pii_types: ["PER", "PHONE_RU"]`
3. В метаданных чанка в Qdrant: `pii_masked: true` — агент может учитывать при ответе
4. При следующем запросе агент сообщает пользователю о найденных и замаскированных данных: «В файле `Регламент_найма.docx` найдены персональные данные (ФИО, телефоны) — они замаскированы перед индексацией»

Сам файл на диске **не изменяется**.

## Производительность

* Модель `ru_core_news_sm` грузится один раз — синглтон как BGE, по той же схеме задачи 9
* Выгружается по таймауту простоя (параметр в конфиге)
* Regex-паттерны отрабатывают первыми (быстро), NER только если regex дал результат или для сущностей типа PER/ORG/LOC
* RAM: \~200-300MB дополнительно

## Конфиг

```yaml
pii_filter:
  enabled: true                    # выключить если не нужно
  scan_user_docs: true             # личные доки
  scan_custom_config: true         # нетиповые объекты XML
  report_to_user: true             # сообщать пользователю о найденном
  spacy_model: ru_core_news_sm     # можно сменить на md/lg
  mask_ner_per: true               # ФИО
  mask_ner_org: false              # организации — спорно, может ломать контекст
  mask_ner_loc: false              # локации — аналогично
  idle_timeout_seconds: 600        # выгрузить модель если нет запросов
```

`mask_ner_org` и `mask_ner_loc` по умолчанию **выключены**: название контрагента или города в регламенте — это часть контекста документа, а не персональные данные. Включать только явно.
