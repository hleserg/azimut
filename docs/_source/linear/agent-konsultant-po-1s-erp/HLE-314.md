Согласовать с контактом в Sentry использование спонсорской/опенсорсной подписки (выданной под **atman**) для НОВОГО опенсорсного проекта (агент-консультант по 1С ERP, лицензия MIT).

## Почему обязательно ДО инструментации

Спонсорские гранты Sentry обычно привязаны к КОНКРЕТНОМУ одобренному проекту/организации. Использование под другой проект — даже опенсорсный с MIT — формально выход за рамки гранта. Биллинг видит события с нового DSN/проекта; могут заметить и отозвать (в т.ч. рискуя грантом под atman). MIT-лицензия — необходимое, но НЕ достаточное условие: грант привязан к тому одобрению, а не к классу «любой мой опенсорс».

## Что сделать

* Написать контакту в Sentry: есть второй опенсорсный проект (MIT, публичный), можно ли покрыть той же спонсорской подпиской / добавить в ту же организацию, или нужна отдельная заявка.
* Зафиксировать ответ.

## Возможные исходы (любой ок)

* «Да, добавь второй проект в ту же орг» → можно делать 0c.
* «Нужна отдельная заявка» → подать, дождаться.
* «Грант только под atman» → для нового проекта либо free-tier Sentry, либо без Sentry / другой инструмент. 0c адаптировать под решение.
