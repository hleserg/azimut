## Что нужно сделать

### 1\. Хранилище реестра

SQLite-база (`registry.db`) рядом с индексами, путь из конфига. Схема:

```sql
CREATE TABLE documents (
    id          INTEGER PRIMARY KEY,
    file_path   TEXT UNIQUE NOT NULL,
    source_type TEXT NOT NULL,  -- 'config_xml' | 'its_docs' | 'user_docs'
    file_hash   TEXT,           -- SHA-256 hex
    file_size   INTEGER,
    last_seen_at     DATETIME,  -- последний раз файл был на диске
    last_indexed_at  DATETIME,  -- последняя УСПЕШНАЯ индексация
    last_attempt_at  DATETIME,  -- последняя ЛЮБАЯ попытка индексации
    index_status TEXT NOT NULL DEFAULT 'pending',
        -- 'pending' | 'ok' | 'error' | 'unsupported' | 'advisory' | 'deleted'
    error_message TEXT,
    chunks_count  INTEGER DEFAULT 0
);
```

### 2\. Статусы

| Статус | Смысл |
| -- | -- |
| `pending` | Новый файл, ещё не обрабатывался |
| `ok` | Проиндексирован успешно |
| `error` | Попытка была, но файл не удалось обработать полностью |
| `unsupported` | Формат не поддерживается |
| `advisory` | Формат поддерживается условно (`.xlsx`) — пользователю предложено конвертировать |
| `deleted` | Файл был в реестре, но исчез с диска |

### 3\. Логика обновления при индексации

```python
def should_reindex(record) -> bool:
    if record.index_status in ('error', 'pending'):
        return True  # всегда перепробуем
    if record.index_status == 'ok':
        current_hash = compute_hash(record.file_path)
        return current_hash != record.file_hash  # перепробуем только если изменился
    return False
```

### 4\. Обработка удалённых файлов

При сканировании папки: если файл в реестре, но на диске его нет:

1. Ставим `index_status = 'deleted'`
2. Запускаем удаление чанков из Qdrant по фильтру `source_path == file_path`
3. Реестр не чистим сразу — помечаем как deleted, чистим при явном `gc()` или через N дней

Поиск в Qdrant ВСЕГДА фильтрует `index_status != 'deleted'` — выдача чистая автоматически.

### 5\. API реестра

```python
class DocumentRegistry:
    def upsert(file_path, source_type, ...) -> None
    def mark_ok(file_path, chunks_count) -> None
    def mark_error(file_path, error_message) -> None
    def mark_deleted(file_path) -> None
    def mark_advisory(file_path, message) -> None
    def get_failed_files(source_type=None) -> List[Record]
    def get_pending_reindex(source_type=None) -> List[Record]
    def get_stats() -> dict  # для healthcheck и MCP get_index_status
    def gc(older_than_days=30) -> int  # удаляет старые deleted записи
```

### 6\. Интеграция с healthcheck

`get_stats()` возвращает по каждому `source_type`:

* всего файлов / ok / error / deleted / unsupported / advisory
* дата последней успешной индексации
* флаг `reindex_needed` (есть файлы с `pending` или изменился хеш)
