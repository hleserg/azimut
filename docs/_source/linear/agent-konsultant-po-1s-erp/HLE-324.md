## Описание

## Цель

Заложить архитектуру, при которой MCP-сервер можно подключить к любому LLM (Claude, OpenAI-совместимые, локальные Ollama, GigaChat и др.) без переписывания бизнес-логики — только добавив адаптер.

## Что нужно сделать

### 1\. Структура папок

```
adapters/
  base.py              # Protocol + AdapterManifest dataclass
  mcp_adapter.py       # Claude Desktop — текущий, реализуется здесь

prompts/
  system_prompt.md     # ОДИН источник правды — поведенческий контракт агента
  tool_hints.md        # подсказки агенту как пользоваться инструментами
```

Папка `adapters/` резервируется сразу. Остальные адаптеры (`openai_adapter.py`, `ollama_adapter.py`, `gigachat_adapter.py`) — заглушки с `raise NotImplementedError` или просто пустые файлы с комментарием.

### 2\. `AdapterManifest` и `MCPAdapter` Protocol

```python
# adapters/base.py
from dataclasses import dataclass
from typing import Protocol

@dataclass
class AdapterManifest:
    system_prompt: str        # поведенческий контракт из prompts/
    tool_definitions: list    # схемы инструментов в формате провайдера
    client_config: dict       # конфиг клиента (claude_desktop_config.json и т.п.)
    install_instructions: str # что сказать пользователю

class MCPAdapter(Protocol):
    def install(self) -> AdapterManifest: ...
```

### 3\. `mcp_adapter.py` — реализация для Claude Desktop

`install()` должен:

* Сгенерировать блок `mcpServers` для `claude_desktop_config.json`
* Найти путь к конфигу платформенно (macOS / Windows)
* Смёрджить (не затереть!) существующий конфиг
* Вернуть `AdapterManifest` с промптом из `prompts/system_prompt.md` и инструкцией «Перезапусти Claude Desktop»

### 4\. `prompts/system_prompt.md`

Черновик поведенческого контракта агента (детальная проработка — в задаче 7, здесь достаточно скелета):

* Агент — консультант по 1С ERP
* Отвечать ТОЛЬКО из результатов MCP-инструментов, не из памяти
* Перечень инструментов и когда их вызывать

### 5\. CLI точка входа

```bash
python -m server install                    # Claude Desktop (дефолт)
python -m server install --adapter openai   # будущий адаптер
python -m server install --dry-run          # показать манифест, ничего не писать
```

В `__main__.py` добавить команду `install` с аргументами `--adapter` и `--dry-run`.

### 6\. Правило в `AGENTS.md`

Добавить раздел **«Адаптеры»**:

* Каждый адаптер в `adapters/` ОБЯЗАН реализовывать `install() -> AdapterManifest`
* Промпт живёт ТОЛЬКО в `prompts/system_prompt.md`, адаптеры его не дублируют
* Бизнес-логика (`tools/`, `indexer/`, `search/`) НЕ импортирует ничего из `anthropic` SDK
* Все I/O инструментов описываются Pydantic-схемами в `schemas/`
