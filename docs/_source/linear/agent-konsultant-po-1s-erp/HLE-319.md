Включить «пройденное ревью Sentry» в обязательную часть проверок PR — наравне с CI и Devin. После этого Sentry-проверка становится частью того «полного зелёного», без которого нельзя мерджить.

## Что сделать

* Добавить проверку Sentry (например, Seer review / Sentry PR checks, как настроено в atman) в набор required-проверок на pull_request.
* Обновить branch protection: Sentry-проверка — обязательная, merge невозможен без её прохождения.
* Обновить git-workflow в `AGENTS.md`: в список «дождаться ВСЕХ проверок и ревьюверов» явно включить Sentry-ревью наравне с Devin и CI. Незакрытая Sentry-проверка = нельзя мерджить.

## Definition of done

* Sentry-проверка в required-проверках PR.
* branch protection не пускает merge без зелёной Sentry-проверки.
* `AGENTS.md` обновлён: Sentry-ревью — обязательная часть цикла «довести до полного зелёного».
* На тестовом PR видно, что Sentry-проверка отрабатывает и блокирует merge при провале.
