Собрать реальные данные ровно в том виде, в каком они будут у конечного пользователя, и зафиксировать пути в едином файле, чтобы агенты точно знали что где лежит.

## Для меня (Сергей) — что собрать и куда положить

Собрать как у пользователя:

* **Выгрузка конфигурации ERP в XML** (Конфигуратор → Выгрузить конфигурацию в файлы) — в отдельную папку.
* **Расширения** (если есть) — выгрузки в XML, каждое в свою папку.
* **Офлайн-ИТС** — развёрнутая офлайн-версия ([its.1c.ru/itsoffline](<http://its.1c.ru/itsoffline>)), путь к папке с htm-материалами.
* **Доки платформы** — если уже добыты с портала техдокументации, путь к ним.

**Куда прописать пути:** создать в корне репо файл `data-paths.local.yaml` (добавить в `.gitignore` — данные и пути вне гита!). Формат:

```yaml
configurations:
  - name: "ERP"
    xml_path: "/абсолютный/путь/к/выгрузке/ERP"
    extensions_paths:
      - "/путь/к/расширению1"
its_docs_path: "/путь/к/офлайн-ИТС"
platform_docs_path: "/путь/к/докам-платформы"  # или null если пока нет
```

Заполнить реальными абсолютными путями на моей машине.

## Для агентов — откуда взять и как обеспечить преемственность

* Пути к данным читать ТОЛЬКО из `data-paths.local.yaml` (он в .gitignore, у каждого свой).
* В репо должен лежать `data-paths.example.yaml` с этой же схемой и комментариями — образец.
* **Преемственность:** первый агент, который реально работает с данными, документирует в `docs/development/DATA_LAYOUT.md`: какая структура у выгрузки XML (по факту увиденного), как выглядит офлайн-ИТС внутри, где что лежит, особенности/подводные камни. Следующие агенты читают этот файл и сразу знают расклад, не переоткрывая.

## Definition of done

* Реальные данные собраны как у пользователя.
* `data-paths.local.yaml` заполнен и лежит в .gitignore.
* `data-paths.example.yaml` (схема + комментарии) — в репо.
* Конфиг проекта (задача 1) умеет читать пути из этого файла.

## Дополнение: третий источник данных — личные документы

При сборе данных зафиксировать также:
- Путь к папке с личными документами пользователя (регламенты, описания доработок, свои процессы). Папка может быть пустой на старте — это нормально.
- Если папки нет или нет желания давать — зафиксировать `user_docs_path: null` и `user_docs_enabled: false`.

Формат в `data_paths.yaml`:
```yaml
config_xml_path: "C:/1C/export/erp/"
its_docs_path: "C:/1C/its_offline/"
user_docs_path: "C:/MyDocs/1c_docs/"   # или null если нет
```

Этот путь используется задачей 20 (парсер личных документов).
