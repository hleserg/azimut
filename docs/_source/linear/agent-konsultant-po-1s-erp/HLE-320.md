## Описание

# \[ТЫ\] Создать шаблон репо (repo-template)

Ручная задача Сергея. Агенты-кодеры её не берут.

## Контекст

Перед тем как агенты начнут работу над следующими проектами, нужен GitHub Template Repository с готовым обвесом. Основа — репо `1c-consultant`, которое Сергей уже собирает. Шаблон превращается в стартовую точку для всех дальнейших проектов.

## Что должно быть в шаблоне

### Агентские правила

* `AGENTS.md` — рабочий протокол агентов-кодеров (взял → In Progress, блокер → стоп, DoD-сверка перед сдачей, In Review, Done ставит Сергей)
* `AGENTS.md` — git-workflow: ветка от main, PR(draft) → снять draft → все проверки зелёные → все замечания resolved → мердж; не вышло → «не могу» + стоп
* Секция про метку `[ТЫ]` — агент не берёт такие задачи

### CI (GitHub Actions)

* Lint (ruff / eslint — по стеку проекта)
* Tests (pytest / jest — по стеку)
* Sentry Release workflow (создание релиза + загрузка source maps / символов)
* Branch protection: мердж только при зелёных проверках + ревью Devin

### Качество кода

* Ruff (Python) — `pyproject.toml` с конфигом
* Pre-commit хуки: ruff, trailing whitespace, end-of-file-fixer, check-yaml, check-toml
* `.editorconfig`
* `commitlint` / conventional commits (опционально, если понравится)

### Тесты

* Структура папки `tests/` с примером теста
* `pytest.ini` или секция в `pyproject.toml`
* Coverage-репорт в CI

### Sentry

* Workflow для создания Sentry Release в CI
* Переменные окружения задокументированы в `.env.example`: `SENTRY_DSN`, `SENTRY_ORG`, `SENTRY_PROJECT`, `SENTRY_AUTH_TOKEN`
* Базовая инициализация Sentry в `src/` (заглушка, раскомментировать по месту)

### Структура проекта (Python-ориентированная)

* `src/<package>/` — основной код
* `tests/` — тесты
* `docs/` — документация
* `reference/` — вендорные спеки/данные (не рантайм-зависимость)
* `scripts/` — утилиты

### Документация

* `README.md` с секциями: описание, быстрый старт, разработка, лицензия
* `LICENSE` (MIT по умолчанию)
* `CHANGELOG.md` (пустой шаблон)
* `docs/ADR/` — папка для Architecture Decision Records

### Конфигурация

* `pyproject.toml` (uv-совместимый)
* `.python-version`
* `.env.example`
* `.gitignore` (Python + uv + IDE + OS)
* `.github/CODEOWNERS` — заглушка

## Definition of Done

- [ ] Репо `hleserg/repo-template` создано на GitHub
- [ ] Отмечено как Template Repository (Settings → Template repository)
- [ ] Все перечисленные файлы на месте и не сломаны
- [ ] CI проходит на пустом (стартовом) состоянии репо
- [ ] Pre-commit хуки работают локально (`pre-commit run --all-files` без ошибок на шаблонном коде)
- [ ] Проверено: создание нового репо из шаблона → CI сразу зелёный
- [ ] Ссылка на `hleserg/repo-template` добавлена в описание этой задачи

## Примечания

* За основу брать актуальное состояние `1c-consultant` — там уже отработаны Sentry, Devin, branch protection, структура
* Шаблон должен быть language-agnostic там где возможно, но Python-first (uv, ruff, pytest)
* Не перегружать: лучше меньше, но всё рабочее из коробки
