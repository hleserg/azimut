## Доставка Sentry DSN (без раскрытия в репо)

DSN разработчика не коммитится в гит. При установке скрипт (`install.sh` / `install.ps1`):

1. Делает запрос к нашему конфигурационному endpoint'у (`https://api.1c-consultant.ru/config` или аналог)
2. Получает JSON с DSN и другими параметрами
3. Кладёт в `~/.1c-consultant/.env` (вне проекта)
4. Sentry SDK при старте читает DSN из `~/.1c-consultant/.env`, не из кода

В репо есть только `.env.example` с `SENTRY_DSN=your_dsn_here` — для разработчиков, которые хотят использовать свой DSN.

**Нет интернета при установке** → DSN не скачивается, Sentry просто молчит (не crash). Код проверяет наличие DSN и грациозно деградирует.

## Consent при первом запуске (GDPR)

При первом запуске агент задаёт **один вопрос** (и только раз):

> «Если что-то пойдёт не так, могу автоматически отправить отчёт об ошибке разработчику — без ваших документов и личных данных, только технические детали. Это помогает быстрее находить и чинить баги. Разрешить?»

* Согласился → `sentry_enabled: true`, `sentry_consent_at: <timestamp>` в конфиг
* Отказался → `sentry_enabled: false`, Sentry инициализируется с no-op transport (не отправляет ничего)
* **Дефолт до ответа: выключено** (GDPR-compliant)

Пользователь в любой момент: «отключи телеметрию» → агент вызывает `update_settings('sentry_enabled', False)`.
