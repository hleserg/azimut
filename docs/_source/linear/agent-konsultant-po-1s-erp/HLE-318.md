После согласования с Sentry (0b) — настроить рабочий конвейер Sentry для проекта и прописать DSN в окружение.

## Что сделать

* Создать проект в Sentry (или добавить в существующую орг — согласно решению 0b).
* Настроить Sentry workflow аналогично atman: проект, алерты, data scrubbing на стороне Sentry, releases (getsentry/action-release в CI), code mappings на GitHub.
* Прописать **DSN в env** (GitHub secrets для CI + локальный `.env`, который в .gitignore). `.env.example` с плейсхолдером DSN — в репо.
* Включить Sentry в CI (релизы, привязка коммитов).

## Definition of done

* Sentry-проект готов на согласованном плане.
* DSN в env (secrets + локально), `.env.example` в репо.
* CI умеет постить релизы в Sentry.
* Разблокированы задачи 0g (Sentry-проверка в PR) и 0c (инструментация).
