Отслеживание новых версий конфигурации ERP и платформы. Изначальное требование «следить за релизами». Низкий приоритет — после рабочего ядра.

## Что сделать

* Шедулер по расписанию: проверять публичные страницы релизов ERP и платформы (портал 1С, новости релизов).
* Детект новой версии → достать changelog, если доступен.
* **Два режима (конфиг):**
  * `notify` — дайджест изменений; агент сообщает при следующем healthcheck/обращении (push невозможен — задача 10).
  * `reindex` — если появилась новая выгрузка/доки под новую версию, подсказать инкрементальную переиндексацию (через задачу 18).
* Легальность: только публичные страницы релизов, без скрейпинга закрытого ИТС.

## Definition of done

* Шедулер находит факт выхода новой версии ERP/платформы.
* Формируется человекочитаемый дайджест.
* Агент сообщает о новой версии при обращении (через healthcheck/статус, не push).
* Режимы notify/reindex переключаются конфигом.
