## Что сделать

* Структура пакета (Python ≥3.11, **управление зависимостями через** `uv`): `src/erp_agent/` с подпакетами `indexing/`, `mcp/`, `sources/`, `config/`.
* `pyproject.toml` с зависимостями (FastMCP, парсер XML, FlagEmbedding для BGE-M3, cohere, qdrant-client). Конкретные версии уточняются в задаче 2. Lock-файл `uv.lock` коммитится в репо.
* Конфиг через файл (`config.yaml`) + переопределение через переменные окружения. В конфиге:
  * список конфигураций: для каждой — `name`, `xml_path`, `extensions_paths`;
  * `its_docs_path`, `platform_docs_path`, `index_path` (всё вне гита);
  * блок эмбеддингов (провайдер: bge-m3 локально по умолчанию / openai опц.; режим гибридный);
  * блок реранкинга (cohere / локальный bge-reranker), ключи Cohere опционально;
  * режим работы (experiment / production), параметры модели (idle_timeout, warm_on_healthcheck).
* `config.example.yaml` в репо с комментариями; реальный `config.yaml` — в `.gitignore`.
* `.gitignore`: реальный конфиг, папка индекса, любые данные, `.env`, логи телеметрии, артефакты.
* `README.md`: что это, как поставить (через `uv sync`), как указать данные, как запустить.

## Definition of done

* `uv sync` ставит зависимости без ошибок; `uv run python -c "import erp_agent"` импортирует пакет.
* Конфиг читается, валидируется (понятная ошибка при кривом конфиге), несколько конфигураций в схеме.
* `uv.lock` закоммичен.
* В гит не попадает ничего из данных пользователя (проверить `git status` с тестовой папкой данных).
