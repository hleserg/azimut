## Поддерживаемые форматы

Все форматы читаются через конкретные библиотеки; нестабильные форматы — не поддерживаются.

| Формат | Библиотека | Примечание |
| -- | -- | -- |
| `.txt`, `.md` | stdlib | Прямое чтение |
| `.pdf` (текст) | `pdfplumber` + `pymupdf` | Основной путь |
| `.pdf` (скан/картинки) | `pytesseract` / `easyocr` | OCR-fallback |
| `.docx` | `python-docx` | Включая картинки внутри |
| `.doc` | LibreOffice headless → конвертация в `.docx` | Конвертируем, затем `python-docx` |
| `.odt` | LibreOffice headless → конвертация в `.docx` | Аналогично `.doc` |
| `.pptx` | `python-pptx` | Текст слайдов + заметки |
| `.csv` | `pandas` | Как plain text с разделителями |
| `.xml` | `lxml` | Парсинг + предупреждение о точности |
| `.json` | stdlib | Как структурированный текст |
| `.html`, `.htm` | `beautifulsoup4` | Извлечение текста |
| `.eml` | stdlib `email` | Письма |

**Форматы advisory (не парсим, но фиксируем в реестре):**

* `.xlsx`, `.xls` — агент сообщает: «Если это регламент — сохрани как `.docx`. Если расчёт — я не аналитик Excel, это за пределами моей специализации 😄»
* `.rtf`, `.msg`, `.doc` (если LibreOffice недоступен) — сообщаем о необходимости конвертации

**Неизвестные расширения:** фиксируем в реестре как `unsupported`, агент при следующей сессии сообщает пользователю имя файла и список поддерживаемых форматов.

## Рекурсивный проход

```python
from pathlib import Path

def scan_directory(root: Path):
    for path in root.rglob('*'):
        if path.is_file():
            yield path
```

Глубина вложенности не ограничена.

## Обработка картинок внутри документов

* **DOCX/ODT**: вырезать embedded images через `python-docx`, прогнать через OCR, текст склеить с основным содержимым документа
* **PDF с картинками**: `pymupdf` рендерит страницы → OCR через `pytesseract` для страниц без текстового слоя

## Healthcheck: мониторинг структуры папки

При каждом healthcheck асинхронно (не блокируя основной поток):

1. Рекурсивно сканировать папку `user_docs_path`
2. Сравнить текущий список файлов + SHA-256 хеши с реестром
3. Если обнаружены новые, изменённые или удалённые файлы:
   * Установить флаг `user_docs_reindex_needed: true` в конфиг/стейт
   * Зафиксировать дату обнаружения изменений
4. Агент при следующем запросе видит флаг через MCP и начинает предлагать переиндексацию

**Нудёж агента при флаге:** ненавязчиво, раз за сессию, не на каждый вопрос. Примерно: «Кстати, в папке с вашими документами появились новые или изменённые файлы. Переиндексировать, чтобы они участвовали в поиске?»

## Настройка через агента

Пользователь может в любой момент:

* Задать или изменить путь: «мои доки теперь в `D:\WorkDocs`» → агент вызывает `update_settings('user_docs_path', ...)` и предлагает сразу запустить индексацию
* Отключить папку: «не нужно читать мои документы» → `user_docs_enabled: false`
* Включить обратно: «всё-таки добавь личные доки» → `user_docs_enabled: true`, спросить путь если не задан

## Предупреждение о конфиденциальности

При активации личных документов + используется облачная модель (Claude API, GPT и др.) — агент **один раз** при настройке произносит спокойное предупреждение:

> «Имей в виду: содержимое твоих документов будет отправляться в облачную языковую модель при ответах на вопросы. Если в документах есть коммерческая тайна или конфиденциальные данные — уточни у руководства, не нарушает ли это политику компании. Я просто на всякий случай предупреждаю.»

Флаг `privacy_warning_acknowledged: true` — не повторять после подтверждения.
При локальной модели — предупреждение не показывать.
