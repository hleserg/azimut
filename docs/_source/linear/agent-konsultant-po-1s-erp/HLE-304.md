Онбординг нового пользователя через MCP-инструменты. Агент ведёт пользователя за руку от «только что установил» до «готово к работе».

## MCP-инструменты онбординга

* `validate_path(path, source_type)` — проверить что путь существует и читаем
* `save_config(key, value)` — сохранить настройку (путь к данным, preferences)
* `get_progress()` — статус текущей индексации (N из M файлов, фаза)
* `healthcheck()` — общий статус системы (индексы, модель, реестр, флаги)
* `get_index_status()` — сводка реестра документов (ok/error/unsupported/advisory по source_type)
* `update_settings(key, value)` — изменить любую настройку в рантайме

## Сценарий онбординга (шаги агента)

1. Спросить путь к XML-выгрузке конфигурации 1С → `validate_path` → `save_config`
2. Спросить путь к офлайн-ИТС → `validate_path` → `save_config`
3. Спросить про личные документы:

   > «Есть ли у тебя папка с собственными регламентами, описаниями доработок или инструкциями? Если да — укажи путь, я смогу искать и по ним. Можно пропустить.»
   * Указал путь → `validate_path` → `save_config(user_docs_path, path)`
   * Пропустил → `save_config(user_docs_enabled, false)` — не спрашиваем повторно
4. **Запрос согласия на телеметрию (обязательный шаг):**

   > «Если что-то пойдёт не так, могу автоматически отправить отчёт об ошибке разработчику — без твоих документов и личных данных, только технические детали. Это поможет быстрее чинить баги. Разрешить?»
   * «Да» → `save_config(sentry_enabled, true)`, `save_config(sentry_consent_date, now)`
   * «Нет» / любой отказ → `save_config(sentry_enabled, false)` — Sentry молчит полностью
   * **Дефолт — выключено.** Sentry не отправляет ничего до явного «да».
5. Запустить индексацию → отображать прогресс через `get_progress()`
6. После завершения — `get_index_status()`, сообщить сводку: что проиндексировано, что не удалось, что в advisory
7. Сообщить что готов к работе

## Управление настройками через агента

Пользователь в любой момент может сказать что-то в духе:

* «Мои личные доки теперь на `D:\docs\`» → агент вызывает `update_settings('user_docs_path', 'D:\\docs\\')`, спрашивает запустить переиндексацию
* «Отключи телеметрию» → `update_settings('sentry_enabled', False)`
* «Добавь папку с личными доками» → возвращаемся к шагу 3, валидируем, сохраняем

Агент принимает естественный язык, маппит на вызовы инструментов — не просит «введи команду».

## Предупреждение о конфиденциальности личных доков

Реализуется в задаче 3a (парсер личных доков), но триггерится через онбординг: при первой активации `user_docs_path` + облачная модель — агент показывает предупреждение один раз.

## Дополнение: онбординг личных доков + телеметрия + коммерческая тайна

По итогам проектирования — в онбординг добавляются три новых момента.

### 1. Вопрос про личные документы

При первом запуске агент спрашивает про третий источник данных:

> «Есть ещё одна возможность: если у тебя есть папка с регламентами, описаниями доработок или другими рабочими документами — я могу их тоже проиндексировать и учитывать при ответах. Хочешь добавить такую папку?»

- «Да» → агент просит путь, запускает механизм задачи 20
- «Нет» / отказ / «потом» → `user_docs_enabled: false`, **больше не предлагать**, пока пользователь сам не попросит
- Пользователь указал путь → перед индексацией проверить пункт 2 (коммерческая тайна)

### 2. Предупреждение о коммерческой тайне (только при облачной модели)

Если активированы личные доки И используется облачная модель — агент говорит **один раз**:

> «Прежде чем начать — имей в виду: фрагменты твоих документов будут отправляться в облако для поиска ответов. Если в них есть коммерческая тайна или конфиденциальная информация — лучше сначала уточни у руководства, можно ли это делать. Продолжить?»

- «Да» / «понял» / «продолжаем» → `privacy_warning_acknowledged: true` + дата, идём дальше
- «Нет» → `user_docs_enabled: false`, не настаиваем
- При локальной модели — вопрос не задаётся вообще

### 3. Вопрос про телеметрию (Sentry)

В конце онбординга, один раз:

> «Если что-то пойдёт не так, могу автоматически отправить отчёт об ошибке разработчику — без ваших документов и личных данных, только технические детали. Это поможет быстрее находить и чинить баги. Разрешить?»

- «Да» → `sentry_enabled: true`, `sentry_consent: "granted"` + дата
- «Нет» / игнор → `sentry_consent: "denied"`, Sentry молчит, **больше не спрашиваем**

### Порядок вопросов в онбординге

1. Путь к выгрузке конфигурации 1С (основной источник)
2. Путь к офлайн-ИТС (документация)
3. Папка с личными документами (опционально, с предупреждением о тайне если облако)
4. Отчёты об ошибках (Sentry opt-in)
