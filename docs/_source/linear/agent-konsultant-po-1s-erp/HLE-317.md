## Описание

## ⚠️ Гибридная задача. Путь к репо прописывает владелец (Сергей), изучение и документирование — агенты.

**Фаза 0.** Делать после 0a (репо). Блокирует задачи 3, 4 (парсеры опираются на спеки 1С) и 0e (сбор данных — формат данных понятен из спек).

---

Склонировать cc-1c-skills локально, дать агентам путь, агенты разбирают репо, **обучаются работать с форматами данных 1С и фиксируют знание для следующих агентов**, и только потом проект опирается на эти спеки в парсерах.

## Для меня (Сергей)

* Склонировать [github.com/Nikolay-Shirokov/cc-1c-skills](<http://github.com/Nikolay-Shirokov/cc-1c-skills>) (MIT) локально.
* Прописать путь в `data-paths.local.yaml` (тот же файл, что в 0e): добавить ключ `cc_1c_skills_path: "/путь/к/cc-1c-skills"`.
* (Можно вместо клона — завендорить нужные `.md`-спеки прямо в репо проекта, см. ниже; тогда путь не нужен.)

## Для агентов — изучить и обучить следующих

1. Разобрать спецификации из cc-1c-skills, ключевые для нашего парсера:
   * `1c-config-objects-spec.md` (23 типа объектов метаданных),
   * `1c-configuration-spec.md` (Configuration.xml, ConfigDumpInfo.xml, 44 типа ChildObjects),
   * `1c-help-spec.md` (встроенная справка Help.xml),
   * `1c-form-spec.md`, `1c-dcs-spec.md` (СКД), `1c-extension-spec.md` (CFE).
2. **Завендорить** нужные спеки в репо проекта в `reference/1c-specs/` (с указанием источника и MIT-лицензии), чтобы агенты читали их с диска, а не зависели от внешнего клона.
3. **Обучить следующих:** написать `docs/development/1C_FORMATS.md` — выжимка «как читать XML-выгрузку 1С»: какие типы объектов, как устроены реквизиты/табличные части/движения/связи, где встроенная справка, на что обращать внимание при парсинге. Это конспект, по которому следующий агент сразу умеет работать с форматом, не перечитывая все спеки заново.

## Definition of done

* Путь к cc-1c-skills прописан (или спеки завендорены).
* Нужные спеки лежат в `reference/1c-specs/` с указанием источника и лицензии.
* `docs/development/1C_FORMATS.md` — рабочий конспект по форматам 1С для следующих агентов.
* Задача 3 (парсер) может опираться на завендоренные спеки с диска.
