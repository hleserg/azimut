## Главное — fallback-цепочка (тема 2)

Официальная формулировка (`tooling-playbooks.md`): `graph → code-metadata → grep=true retry → Grep`. Только для поиска по исходникам проекта (не для внешних знаний).

1. **Граф** (Neo4j/Cypher) — высший приоритет.
2. **code-metadata** (FTS+semantic) — когда граф недоступен или вернул пусто/нерелевантно.
3. **code-metadata + `grep=true`** — substring-retry внутри MCP-индекса (НЕ файловый grep): точные идентификаторы, фрагменты, пути метаданных, имена обработчиков, текст ошибок.
4. **файловый `Grep`/`rg`** — только когда MCP исчерпан, с обязательной пометкой что пробовал.

**Триггер перехода вниз:** пусто/non-actionable **дважды на существенно разных запросах** по одной цели (не с первой неудачи). Принцип остановки: стоп когда данных достаточно; no-change repeats запрещены.

## Диспетчер — два вектора

- Поиск по проекту — каскад выше.
- Внешние знания (платформа/БСП/шаблоны/живая ИБ) — без grep-эквивалента, по нужде.

Сервер доступен только если инструменты реально экспонированы в сессии (наличие в конфиге ≠ доступность).
