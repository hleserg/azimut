* Как решают проблему ОДНОИМЁННЫХ процедур в разных модулях? Хранят ли квалификатор (`ОбщийМодуль.Метод`), тип вызова, область видимости? Есть ли алгоритм разрешения имени → конкретный узел?

## Рекомендация по схеме графа для Азимута

**Схема метаданных** → брать из **bsl-graph (MIT)**:
- `NodeType` enum (MDObjectNode.kt:26-106) — 1:1 маппинг MDOType, полный
- `ATTRIBUTE` ребро с property name — элегантно
- `ACCESS` ребро с правами
- Логика MetadataExporter.kt — переиспользовать напрямую
- `MDOReader.readConfiguration()` как точка входа

**Схема кода** → добавить по образцу **1c-mcp-metacode**:
- Узел `Module` (OBJECT_MODULE, MANAGER_MODULE, FORM_MODULE, COMMON_MODULE)
- Узел `Routine` (procedure/function) с привязкой к Module
- `Module →[DECLARES]→ Routine`
- `MetadataObject →[HAS_MODULE]→ Module`
- `Routine →[CALLS]→ Routine` (resolved ссылка!)

**Подписки** → схема metacode: `EventSubscription →[HAS_HANDLER]→ Routine`

**Граф. БД:**
- На старте (только метаданные): **SQLite достаточно**. ~5K узлов, ~20K рёбер — BFS в коде.
