## Контекст

Это **сердце проекта**: агент должен отвечать только из найденного и честно говорить «не нашёл», а не сочинять. Причины галлюцинаций при ХОРОШИХ данных: (1) внутреннее знание перебивает контекст (Claude видел тонны ТИПОВОЙ 1С, тянет привычное вместо кастомного); (2) чанк неудобно собран → модель «дорисовывает»; (3) соблазн полноты — чанк отвечает на 80%, модель достраивает 20% из головы (самый коварный). Уже принято Р2, Р3, Р5, Р6, Р7; предложены П1, П3; открыт Р1.

## Вопросы темы

### Финализировать принятые направления

* **П1 — детектор «relevance высокая / groundedness низкая», 3 слоя:** (1) судья режет ответ ДО выдачи, ниже порога → возврат агенту «перепиши строго по чанкам или скажи что не нашёл» (стенка, не шлепок); (2) после переписи всё равно низко → плашка «частично из общих знаний, в источниках не подтверждено»; (3) канарейка доли негрунтованного в Sentry. Подтвердить 3 слоя и пороги.
* **П3 — оценка достаточности запроса + подсказки агенту из индекса:** (а) размытый вопрос не молотить поиском вхолостую → «слишком общий»; (б) намёки агенту что переспросить на основе РЕАЛЬНОГО индекса (агент сам не знает, что в данных); (в) проверка, что переформулированный запрос не уехал от исходного. Спроектировать.
* **Р1 — метрика противоречивости (открыто):** как технически детектировать противоречие извлечённых чанков, порог, поведение при множестве конфликтов. Связано с Р6 (иерархия источников: не выкидываем — объясняем по приоритету, помечаем первый как вероятный).

### Поведенческий контракт (был задачей 7 — «сердце»)

* **Заземлённый промпт как последний рубеж:** если Faithfulness < 0,80 даже с реранком — переписать системный промпт с мягкого («помоги по документации») на строгий: только из приведённых фрагментов; цитировать `chunk_id` в каждом утверждении; нет источника → «в документации не описано», додумывать запрещено; разделять «найдено / выведено». ВАЖНО: лечит «модель врёт при наличии источников», НЕ «источники не нашлись» — сначала чинить Context Recall (тема 3/6), потом промпт.
* **Цитирование:** `LlamaIndex CitationQueryEngine` (MIT) — ответы со встроенными `[chunk_id]`; CITATION_QA_TEMPLATE перевести на русский.
* **Навык «это не в коде, это в данных/настройке»** (из эталонного кейса про проводку, тема 2): без него агент галлюцинирует объяснение из кода там, где причина в настройке/регистре/расширении. Отдельный пункт контракта.
* **Фолбэк Р7 в контракте:** «не нашёл в индексе» → дип-ресёрч в интернете С ТЕМ ЖЕ контрактом (показать источник, пометить «из интернета, не из базы», не выдавать за истину). Интернет по 1С — минное поле.

### Кандидат на этап 2

* **Self-RAG** (токены рефлексии `[ISREL]`/`[ISSUP]`/`[ISUSE]` перед и после генерации) — НЕ дообучать модель, а заимствовать паттерн как Claude-промпт-сигналы (`AkariAsai/self-rag`, MIT). По бенчам снижает галлюцинации до \~5,8% против 12–14%. Решать после baseline.
