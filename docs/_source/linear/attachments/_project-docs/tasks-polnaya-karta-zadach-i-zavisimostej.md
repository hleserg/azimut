## Document Registry (задача 19) — API-шпаргалка

```python
class DocumentRegistry:
    def upsert(file_path, source_type) -> None
    def mark_ok(file_path, chunks_count) -> None
    def mark_error(file_path, error_message) -> None   # не падаем, продолжаем
    def mark_deleted(file_path) -> None                # → чистит чанки из Qdrant
    def mark_advisory(file_path, message) -> None
    def get_failed_files(source_type=None) -> List[Record]
    def get_pending_reindex(source_type=None) -> List[Record]
    def get_stats() -> dict                            # для healthcheck + Sentry
    def gc(older_than_days=30) -> int
```

Статусы: `pending` | `ok` | `error` | `unsupported` | `advisory` | `deleted`

Поиск в Qdrant всегда фильтрует `deleted` — пользователь не видит удалённое.

## Форматы личных документов (задача 20)

**Читаем:** `.txt` `.md` `.pdf` `.docx` `.doc`\* `.odt`\* `.pptx` `.csv` `.xml` `.json` `.html` `.htm` `.eml`

**Advisory — не парсим, советуем конвертировать в .docx:** `.xlsx` `.xls`

**Unsupported — сообщаем + список поддерживаемых:** всё остальное

\*требует LibreOffice headless (системная зависимость, не pip — указать в README)

## Sentry — правила конфиденциальности

Инициализируется только при `sentry_enabled: true` (дефолт: **false**, до явного согласия пользователя).

**Никогда не логировать:** содержимое чанков/документов, пути к файлам пользователя, тексты запросов, IP (`send_default_pii=False`).

**Можно логировать:** тип ошибки, стек трейс без данных пользователя, метрики (латентность, RAM, статусы индекса).

DSN не хранится в репозитории — доставляется скриптом установки через наш endpoint.
