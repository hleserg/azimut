## 2. Группа A: детальный разбор

### 2.1 SteelMorgan/1c-log-checker

**Лицензия:** MIT, Акимов Владимир 2025.

**Механизм доступа:** прямое чтение бинарных файлов журналов с файловой системы.  
- Журнал регистрации: `.lgf` (заголовок) + `.lgp` (записи) — Go-структура `Reader` (`internal/logreader/eventlog/reader.go:35-60`).  
- Технологический журнал: `.log` — интерфейс `TechLogReader` (`internal/logreader/interface.go:25-36`).  
- Файлы хранятся там, где их пишет 1С-сервер; в Docker-Compose пробрасываются через volume mount.  
- Прогресс чтения сохраняется в BoltDB (offsets по файлу).

**Хранилище:** ClickHouse, таблица `logs.event_log`. Запрос `internal/handlers/event_log.go:150-160`:
```sql
SELECT event_time, level, event_presentation, user_name, comment, metadata_presentation
FROM logs.event_log WHERE cluster_guid = ? AND infobase_guid = ?
```

**MCP-инструменты (8):** `logc_get_event_log`, `logc_get_tech_log`, `logc_get_actual_log_timestamp`, `logc_configure_techlog`, `logc_save_techlog`, `logc_restore_techlog`, `logc_disable_techlog`, `logc_get_techlog_config`.

**Идентификация базы:** `configs/cluster_map.yaml` — GUID кластера и инфобазы с human-readable именами. MCP-описание инструктирует агента прочитать файл перед вызовом (`internal/mcp/tools.json:5`).

**Особенности:**
- Данные всегда актуальные — Go-парсер читает файлы в реальном времени.
- Два типа журналов: Журнал регистрации (бизнес-события) и Технологический журнал (SQL, блокировки, EXCP).
- Уровни событий поддерживаются на рус. и англ.: `internal/handlers/event_log.go:34-47`.

### 2.2 vladimir-kharin/1c_mcp

**Лицензия:** MIT.

**Механизм доступа:** BSL-расширение (.cfe) устанавливается в живую 1С-базу и регистрирует HTTP-сервис `mcp_APIBackend`. BSL-код выполняется внутри 1С-сервера.  
- Python-прокси (опц.): `httpx.BasicAuth` + POST `{base_url}/hs/mcp/rpc` (`src/py_server/onec_client.py:35,100-116`).  
- Маршрутизация: `HTTPServices/mcp_APIBackend/Ext/Module.bsl:82-96` обрабатывает `tools/list`, `tools/call`, `resources/list`, `resources/read`, `prompts/list`, `prompts/get`.  
- BSL обращается к `Метаданные[МетаТип]` — глобальный объект метаданных текущей базы.

**MCP-инструменты (2, встроенные):**
- `list_metadata_objects` — `Метаданные[metaType]` — список объектов (Справочники, Документы, РегистрыСведений, ..., всего 40+ типов) с фильтром по имени/синониму (`ManagerModule.bsl:78-110`).
- `get_metadata_structure` — `КоллекцияМД.Найти(name)` — реквизиты, табличные части, измерения, ресурсы объекта (`ManagerModule.bsl:156-178`).

**Расширяемость:** любая новая обработка, включённая в подсистему `mcp_КонтейнерыИнструментов`, автоматически становится MCP-инструментом (`agents.md:163-174`). Кастомные инструменты могут обращаться к произвольным данным базы.

**Особенности:**
- Прокси решает проблему аутентификации (OAuth2 → Basic Auth) и stdio-транспорта.
- Расширение НЕ требует изменения конфигурации базы.

### 2.3 ROCTUP/1c-mcp-toolkit

**Лицензия:** файл LICENSE отсутствует. Подходы изучены, код не копирую.

**Механизм доступа:** внешняя обработка (EPF) работает в сессии 1С-клиента — имеет полный доступ к данным и метаданным текущей базы. Два режима:  
- **Встроенный HTTP-сервер**: нативный C++ компонент `MCPHttpTransport` поднимает HTTP-сервер прямо внутри EPF (порт 6003 по умолчанию).  
- **Прокси-режим**: Python FastAPI + long polling (`/1c/poll`, `/1c/result`).

**Ключевые точки BSL** (`1c/MCPToolkit/MCPToolkit/Forms/Форма/Ext/Form/Module.bsl`):
- `ВыполнитьЗапрос()` → `Новый Запрос(ТекстЗапроса)` + `Запрос.Выполнить()` (строки 2737-2770) — язык запросов 1С, произвольный доступ к любым данным базы.
- `ВыполнитьКод()` → `Выполнить(Код)` (строка 3020) — исполнение произвольного BSL.
- `ПолучитьЖурналРегистрации()` → `ВыгрузитьЖурналРегистрации(ТаблицаЖурнала, Отбор, ...)` (строка 7043) — встроенная BSL-функция, читает ЖР текущего сеанса.

**Полный список MCP-инструментов (13):**
`execute_query`, `execute_code`, `get_metadata`, `get_event_log`, `get_object_by_link`, `get_link_of_object`, `find_references_to_object`, `get_access_rights`, `get_bsl_syntax_help` (нативный SyntaxHelpReader), `get_screenshot` (нативный ScreenCapture, Windows), `submit_for_deanonymization`, `restart_1c_session`, `close_1c_session`.

**Особенности:**
- `execute_query` + `execute_code` дают доступ к ЛЮБЫМ данным базы.
- Анонимизация: замена ПДн токенами `[ORG-00001]`, `[PER-00001]`, `[INN-00001]` и т.д.
- Мощнейший инструмент из пяти — полный доступ к данным + метаданным + журналу.
- Не требует изменения конфигурации (`README.md:12`).

## 3. Группа B: краткий разбор

### 3.1 SteelMorgan/spring-mcp-1c-copilot

**Лицензия:** Custom Open Source, некоммерческое использование свободно.

**Суть:** Kotlin/Spring Boot MCP-прокси к облачному AI-сервису **1С:Напарник** (`https://code.1c.ai`). Spring `WebClient` создаёт сессии и отправляет вопросы по API (`/chat_api/v1/conversations/`). Ответ — SSE-поток.

**MCP-инструменты:** `ask_1c_ai`, `explain_1c_syntax`, `check_1c_code` — все обращаются к внешнему AI, данные конкретной базы не затрагиваются.

**Вывод:** не релевантно «теме 7» — нет доступа к runtime-данным конкретной 1С-базы.

---

### 3.2 SteelMorgan/1c-ai-sandbox-client-server

**Лицензия:** MIT, Акимов Владимир 2026.

**Суть:** инфраструктурная песочница. Клиентский Dev Container (Linux + 1С-клиент + Xvfb + Claude Code/Codex/Gemini CLI) + серверная Hyper-V VM (Docker Compose: `onec-server` + `postgres` + `onec-web`). Собственных MCP-инструментов нет.

**Вывод:** не релевантно «теме 7» как самостоятельный инструмент. Служит платформой для запуска других инструментов (например, 1c-mcp-toolkit).

## 4. Сравнительный анализ Group A

| Параметр | 1c-log-checker | 1c_mcp | 1c-mcp-toolkit |
|---|---|---|---|
| **Транспорт** | Файлы ФС (lgf/lgp/log) | 1С HTTP-сервис | EPF в сессии + HTTP или long polling |
| **Установка в 1С** | Не требуется | Расширение (.cfe) | Открыть EPF в клиенте |
| **Данные** | Только журналы | Только метаданные + custom | Любые данные (execute_query) |
| **Доступ к бизнес-данным** | Нет | Опционально (кастомные инструменты) | Да (execute_query, execute_code) |
| **Журнал регистрации** | ClickHouse | Нет | Да (ВыгрузитьЖурналРегистрации) |
| **Техжурнал** | ClickHouse | Нет | Нет |
| **Метаданные конфигурации** | Нет | Да (list_metadata_objects, get_metadata_structure) | Да (get_metadata) |
| **Безопасность** | Только чтение | Зависит от прав 1С-пользователя | execute_code = полный доступ; есть черный список + режим подтверждения |
| **Анонимизация ПДн** | Нет | Нет | Да (опциональная, токенизация) |
| **Лицензия** | MIT | MIT | Нет лицензии |
| **Стек** | Go | BSL + Python | BSL + Python + C++ |
| **Сложность развертывания** | Средняя (Docker + ClickHouse) | Низкая (EPF/расширение) | Средняя (EPF + опц. Python) |

## 5. Ключевые наблюдения для проекта «Азимут»

1. **Три разных уровня доступа**:
   - 1c-log-checker — только логи (ЖР + ТЖ), нет доступа к данным приложения.
   - 1c_mcp — только метаданные конфигурации (структура, не данные), расширяем кастомными инструментами.
   - 1c-mcp-toolkit — полный доступ (execute_query + execute_code + ЖР + метаданные).

2. **Механизмы доступа**:
   - Файловая система (1c-log-checker) — самый низкоуровневый, не требует ничего устанавливать в 1С.
   - 1С HTTP-сервис (1c_mcp) — стандартный способ публикации 1С-функциональности, но требует расширения.
   - BSL через EPF (1c-mcp-toolkit) — работает внутри живого сеанса 1С-клиента, максимальная мощь.

3. **Для «Азимута»** (анализ кода и метаданных конфигурации): наиболее близкий по задаче — 1c_mcp (доступ к метаданным через BSL, аналог того, что делают анализируемые нами граф-репы, но в runtime). Паттерн «1С HTTP-сервис + BSL-код внутри 1С» — это тот механизм, который позволяет AI-агенту взаимодействовать с живой конфигурацией без внешних инструментов разбора XML.

4. **1c-mcp-toolkit/execute_query** — ближайший аналог того, что 1c-mcp-metacode делает через Cypher в Neo4j: произвольные запросы к данным живой базы. Но в toolkit нет граф-модели — только табличные запросы.

5. **Отсутствие лицензии у 1c-mcp-toolkit** — критично для коммерческого использования. Перед любым копированием кода нужен явный ответ автора.
