### #8 — Слепые зоны bsl-atlas: «частично» по подпискам, а не «полная слепая зона»

| | |
|---|---|
| **Что было** | В HLE-494 подписки на события перечислены как одна из «слепых зон bsl-atlas». В самом источнике точнее: тип объекта `ПодпискаНаСобытие` индексируется (тип виден), а кросс-объектные рёбра «событие → обработчик» НЕ строятся. Разное по смыслу для проектирования. |
| **Решение** | **Использовать формулировку «частично»** во всех новых документах: «частичная слепая зона — тип объекта `ПодпискаНаСобытие` индексируется, рёбра `событие → обработчик` достраиваем сами (поверх подхода metacode)». Архитектор сразу видит «допиливать», а не «строить с нуля». |
| **Статус** | `done` (формулировка зафиксирована тут; в `design-system-v2.md` раздел «📂 Что bsl-atlas реально делает» уже описывает это корректно — точечной правки не потребовалось). |
