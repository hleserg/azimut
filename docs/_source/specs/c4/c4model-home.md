## Overview

The C4 model is described as "an easy to learn, developer friendly approach to software" diagramming that employs hierarchical levels for architectural documentation.

## Core Components

### Four Hierarchical Abstractions

1. **Software Systems** — the highest level
2. **Containers** — decomposition within systems
3. **Components** — decomposition within containers
4. **Code** — the lowest level of detail

### Four Hierarchical Diagrams (corresponding to each abstraction level)

1. System context diagram
2. Container diagram
3. Component diagram
4. Code diagram

## Supporting Diagram Types

Beyond the core four levels, the C4 model includes:
- System landscape diagram
- Dynamic diagram
- Deployment diagram

## Key Characteristics

- **Notation independent** — not tied to specific diagramming standards
- **Tooling independent** — can be implemented with various tools
- Designed for developer audiences
- Built on "hierarchical abstractions" and corresponding "hierarchical diagrams"

## Key Quote from Site

> "The different levels of zoom allow you to tell different stories to different audiences."
>
> "You don't need to use all 4 levels of diagram; only those that add value."
