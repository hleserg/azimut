# 2. Архитектурные ограничения (Architecture Constraints)

> arc42 §2 — технические и организационные ограничения, которые не пересматриваются в рамках обычного дизайна. Любое из них снимается только отдельным ADR со статусом `supersedes`.
>
> Источники: [`_source/notion/design-system-v2--*.md`](https://github.com/hleserg/azimut/blob/master/docs/_source/notion/design-system-v2--36b0c905e626813491fcf7e9ccf2046e.md) раздел «Что НЕ меняется», [`_source/notion/hub-1c-assistent--*.md`](https://github.com/hleserg/azimut/blob/master/docs/_source/notion/hub-1c-assistent--36b0c905e626819eaca7ed0875cb6f8e.md), `LICENSE` (AGPL-3.0) в корне репо.

Причина — единственный способ дёргать JVM из Python и сохранить «маме не ставить вторую среду». | [ADR 0011](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0011-fork-bsl-atlas-as-core.md), [ADR 0023](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0023-license-checklist-and-source-rule.md) |

## 2.2 Организационные ограничения

| ID | Ограничение | Откуда / чем зафиксировано |
|---|---|---|
| O1 | **Правило источников.** Любое утверждение о лицензии/версии/внешнем факте помечается: «✅ проверено: \<файл/url\>» либо «⚠️ предположение, не проверено». Без пометки факт в источники истины не пишется. Причина — в ранних заходах лицензия `bsl-atlas` и ряда библиотек была записана по памяти (галлюцинация) и оказалась неверной. Это ровно тот класс ошибок, против которого строится Азимут. | [ADR 0023](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0023-license-checklist-and-source-rule.md); [`_source/notion/design-system-v2--*.md`](https://github.com/hleserg/azimut/blob/master/docs/_source/notion/design-system-v2--36b0c905e626813491fcf7e9ccf2046e.md) реш. 1.10 |
| O2 | **Правило про код 1С (сквозное).** Ассистент НЕ придумывает паттерны, имена событий, поведение платформы или «как обычно пишут в 1С». Любое утверждение про код 1С — либо подтверждённый факт из исходников (анализ Claude Code / прогон / реальная выгрузка), либо явно согласовано с Сергеем. Где факта нет — пишем «не знаю, надо проверить». Применяется и к коду самого Азимута, и к ответам пользователю. | [`_source/notion/design-system-v2--*.md`](https://github.com/hleserg/azimut/blob/master/docs/_source/notion/design-system-v2--36b0c905e626813491fcf7e9ccf2046e.md) §«ЖЁСТКОЕ ПРАВИЛО ПРО КОД 1С» |
| O3 | **OSS-цель важнее монетизации.** Проект открытый под AGPL-3.0; монетизация-через-закрытость не планируется. Если деньги появятся — через услуги (внедрение/настройка/поддержка) поверх открытого кода. Решения «закрыть исходник ради конкурентного преимущества» — отклоняются. | [`_source/notion/design-system-v2--*.md`](https://github.com/hleserg/azimut/blob/master/docs/_source/notion/design-system-v2--36b0c905e626813491fcf7e9ccf2046e.md) §«Что НЕ меняется»; [ADR 0011](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0011-fork-bsl-atlas-as-core.md) |
| O4 | **GDPR-флоу для телеметрии.** Любая канарейка/мониторинг (Sentry / GlitchTip / self-host) — с opt-in, маска PII перед отправкой, документированный data flow. Подробности — в [`08-cross-cutting-concepts.md`](https://github.com/hleserg/azimut/blob/master/docs/architecture/08-cross-cutting-concepts.md) §«Приватность и телеметрия». | [ADR 0023](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0023-license-checklist-and-source-rule.md); [ADR 0028](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/open/0028-sentry-vs-agpl.md) (Sentry × AGPL — open) |
| O5 | **152-ФЗ для коммерческой эксплуатации.** Если/когда коммерческие тенанты — обработка ПДн только провайдерами с законной локализацией (YandexGPT/GigaChat как комплаенс-фолбэк). Прототип + локальное использование Сергея/мамы + маска PII — out of scope 152-ФЗ. Развилка фиксируется в теме 7. | [ADR 0020](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0020-cloud-llm-via-adapter.md), [ADR 0021](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0021-default-model-deepseek-v4.md); тема 7 — [HLE-419](https://linear.app/hleserg/issue/HLE-419) |
| O6 | **Definition of Done (см. [`AGENTS.md`](https://github.com/hleserg/azimut/blob/master/AGENTS.md) §6).** Задача не закрыта, если не обновлены `workspace.dsl` (при изменении архитектуры), ADR (любое архитектурное решение), глава 13 (при добавлении инфраструктурного элемента) и индекс ADR (`./scripts/update-adr-index.sh`). Это организационное, а не техническое — нарушается коммитом, ловится ревью. | [`AGENTS.md`](https://github.com/hleserg/azimut/blob/master/AGENTS.md) §6; [`13-lead-operating-manual.md`](https://github.com/hleserg/azimut/blob/master/docs/architecture/13-lead-operating-manual.md) |
| O7 | **Статус-протокол агентов.** При взятии задачи Linear-тикет переводится в `In Progress`; при готовности PR — в `In Review`. `Done` ставит только Сергей. Master-агент обновляет статус оперативно. | [`AGENTS.md`](https://github.com/hleserg/azimut/blob/master/AGENTS.md) §10 |

## 2.3 Что выходит из ограничений

- **Sentry × AGPL** — Sentry for Open Source просит permissive-лицензию (Apache/MIT), а мы AGPL. Открытый вопрос; план Б — GlitchTip / self-host. Форк bsl-atlas НЕ переоткрывается. См. [ADR 0028](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/open/0028-sentry-vs-agpl.md).
- **Мульти-аренда и веб-морда** — четыре развилки ([ADR 0029](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/open/0029-multitenancy-qdrant-embedded-vs-server.md), [ADR 0030](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/open/0030-multitenancy-canary-vs-watchdog.md), [ADR 0031](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/open/0031-multitenancy-push-via-web-frontend.md), [ADR 0032](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/open/0032-multitenancy-tenant-storage-isolation.md)) — закрываются при работе над темой 7.
- **Sentry-грант (\$1000)** — рычаг Сергея, не автоматическая зависимость пользователя; ничего не «отваливается», если грант не использовать. См. [ADR 0023](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0023-license-checklist-and-source-rule.md).
- **Технический риск дымового прогона форка на реальном ERP** — насколько `bsl-atlas` сцеплен со своим хранилищем (ChromaDB), миграция на Qdrant дороже, чем кажется. Подробности — в [`11-technical-risks.md`](https://github.com/hleserg/azimut/blob/master/docs/architecture/11-technical-risks.md), решение по миграции — [ADR 0015](https://github.com/hleserg/azimut/blob/master/docs/architecture/adr/foundation/0015-stack-migration-smoke-then-qdrant.md).
