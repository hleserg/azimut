# Фолбэк = смена режима (дип-ресёрч в интернете с тем же контрактом)

## Context and Problem Statement

Что делать, если ретривер по локальному индексу ничего полезного не вернул? Исходное Р4 (2026-05-25, см. ADR 0004) предлагало «честно ответить не знаю». Это превращает бота в «обезьянку из колл-центра, копай сам» — бесполезно. Нужна альтернатива, которая сохраняет ценность для пользователя и при этом не нарушает контракта «не выдавать за истину».

## Decision Drivers

* Польза для пользователя: даже при пустом локальном индексе у нас есть следующий шаг — дип-ресёрч в интернете.
* Сохранение контракта на честность источников: «из интернета» должно быть видно в ответе и не подаваться как факт из базы.
* Безопасность: интернет по 1С — минное поле (форумы, устаревшие версии, советы под другую редакцию). Контракт «показать источник, пометить как внешний» должен работать там же, где и для локального индекса.
* Совместимость с остальными принципами (Р3 — судья со спан-привязкой, ADR 0003; Р6 — иерархия, ADR 0006). Дип-ресёрч — отдельный «нижний» уровень в иерархии.

## Considered Options

* (A) Фолбэк = переход в режим дип-ресёрча в интернете с явной пометкой источника (Р7).
* (B) «Честный тупик» — ответить «не нашёл» и не отвечать дальше (исходное Р4, см. ADR 0004).
* (C) Любые догадки от модели из её внутреннего знания.

## Decision Outcome

Chosen option: «(A) смена режима в дип-ресёрч с контрактом источников», because:

* (B) бесполезно для пользователя в ценной ситуации, когда у него нет другого источника.
* (C) ровно та галлюцинация, ради борьбы с которой существуют Р2, Р3, П1.
* (A) даёт пользу, сохраняет контракт и наблюдаемо разделяет «из базы» / «из интернета».

### Consequences

* Good, because пользователь получает следующий шаг, а не отказ.
* Good, because источник всегда виден — снимает риск подачи интернет-результата как «факта из базы».
* Bad, because дип-ресёрч сам по себе шумный (форумы, устаревшие версии); контракт «помечать источник» обязателен и не предмет компромисса.
* Bad, because стоимость и латентность фолбэк-вызовов выше базового сценария.

### Confirmation

* В `06-runtime-view.md` зафиксирован отдельный сценарий «фолбэк → дип-ресёрч в интернете».
* В ответах фолбэк-режима всегда есть пометка «из интернета, не из локального индекса».
* В Sentry метрика `fallback_triggered` отделена от обычного `low_relevance`.

## Pros and Cons of the Options

### (A) Смена режима

* Good, because польза для пользователя сохраняется.
* Good, because контракт честности соблюдается через явную пометку.
* Bad, because дороже и шумнее, нужны фильтры.

### (B) Честный тупик (Р4)

* Good, because безопасно: модель ничего не дорисовывает.
* Bad, because для пользователя — отказ без альтернативы.

### (C) Догадки модели

* Good, because дёшево.
* Bad, because галлюцинация по определению (внутреннее знание, не подкреплено).
