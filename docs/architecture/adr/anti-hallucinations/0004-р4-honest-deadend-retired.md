# «Честный тупик» как фолбэк (снято)

## Context and Problem Statement

Изначально (Р4, 2026-05-25) в `_source/notion/decisions--*.md` обсуждался вариант фолбэка: если ретривер ничего не нашёл — агент честно отвечает «не знаю / в индексе нет / разбирайся сам». Решение зафиксировано в этом ADR как **надгробие**, чтобы при чтении папки `anti-hallucinations/` не возникало впечатления, что Р4 «потерялся».

## Decision Drivers

История фиксируется для прослеживаемости. Аргументы за снятие:

* «Копай сам» = ответ из колл-центра обезьянкой; для пользователя бот в этом случае бесполезен.
* Контракт «не нашёл — переход в другой режим» (Р7 / ADR 0007) сохраняет ценность для пользователя и при этом не нарушает других принципов (источник помечается «из интернета», не подаётся как истина).

## Considered Options

* (A) Оставить «честный тупик» как фолбэк (исходный Р4).
* (B) Заменить на смену режима в дип-ресёрч (Р7 / ADR 0007).

## Decision Outcome

Chosen option: «(B) Заменить на Р7», 2026-05-26. См. ADR 0007.

### Consequences

* Good, because пользователь получает следующий шаг, а не отказ.
* Bad, because дип-ресёрч по 1С — минное поле, контракт «помечать источник» становится особенно важным (см. ADR 0007).

### Confirmation

* В коде/доке нет ветки «вернуть пустой ответ как финал». Все no-result-ветки уходят в дип-ресёрч-режим.
* В `06-runtime-view.md` сценарий «не нашёл» ведёт в фолбэк-режим, а не в финальный отказ.
