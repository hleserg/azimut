# Детектор «relevance высокий / groundedness низкий» — 3 уровня действий

## Context and Problem Statement

Самый коварный класс галлюцинаций: ретривер нашёл релевантные чанки (`relevance` высокий), но ответ опирается на них слабо (`groundedness` низкий) — модель «дорисовала полноту», поскольку чанки покрыли 80% и оставшиеся 20% соблазнительно достроить из внутреннего знания. Нужен механизм с тремя уровнями реакции в зависимости от того, насколько серьёзно расхождение.

## Decision Drivers

* Гранулярность реакции: одно «отказать» на любую недогрунтованность — это слишком грубо; терпеть всё — это П1 не выполняет.
* Совместимость с Р3 (LLM-судья, ADR 0003): уровни должны быть надстройкой над сигналом судьи, а не самостоятельным детектором.
* Видимость метрики (Sentry): доля негрунтованных ответов = калибруемый показатель.
* Защита пользователя: пограничные ответы должны идти с явной плашкой, а не маскироваться под уверенный ответ.

## Considered Options

* (A) Три уровня действий поверх судьи:
  * Уровень 1 («стенка»): ответ ниже жёсткого порога — НЕ отдаётся, агенту возврат «перепиши строго по чанкам или скажи, что не нашёл».
  * Уровень 2 («плашка»): после переписи всё ещё ниже мягкого порога — ответ отдаётся с пометкой «частично из общих знаний, в источниках не подтверждено».
  * Уровень 3 («канарейка»): доля негрунтованных ответов копится в Sentry — тренд для калибровки.
* (B) Один порог: ниже — блок, выше — пропуск.
* (C) Только канарейка (логировать, не блокировать).

## Decision Outcome

Chosen option: «(A) три уровня», because:

* (B) теряет полутень: ответ может быть «местами недогрунтован» — блокировать его жёстко = потеря пользы; пропускать — потеря контракта.
* (C) превращает П1 в наблюдение без действия — это уже частично делает Р2 (ADR 0002, метрики).
* (A) сохраняет полезный ответ с честной плашкой и одновременно жёстко закрывает «совсем плохие».

Статус оставлен `proposed`: конкретные пороги уровня 1 / уровня 2 не зафиксированы — нужна калибровка на пилотном датасете.

### Consequences

* Good, because пользователь не получает уверенно-выглядящий негрунтованный ответ молча.
* Good, because есть путь «дать ответ с плашкой» — польза не теряется в средних случаях.
* Bad, because требует двух порогов и их калибровки на реальных данных — открытый риск.
* Bad, because уровень 1 (блок + возврат агенту на переписи) даёт дополнительный LLM-вызов — латентность.

### Confirmation

* В пайплайне ответа есть три ветки реакции на сигнал судьи: блок-и-возврат / плашка / лог.
* В Sentry метрики `groundedness_below_hard_threshold`, `groundedness_below_soft_threshold` ведутся раздельно.
* На пилотном датасете подобраны пороги, при которых уровень 1 ловит явные дорисовки (precision ≥ 0.9 — целевая планка для калибровки).

## Pros and Cons of the Options

### (A) Три уровня

* Good, because гранулярная реакция.
* Good, because плашка сохраняет пользу.
* Bad, because нужны два порога и их калибровка.

### (B) Один порог

* Good, because просто.
* Bad, because теряет полутень.

### (C) Только канарейка

* Good, because дёшево и не ломает выдачу.
* Bad, because П1 без действия = просто Р2 в другой обёртке.
